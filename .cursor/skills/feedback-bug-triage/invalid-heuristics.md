# 无效 / 低质量 bug 判定细则

Agent 扫描 `category === "bug"` 时使用。满足 **任一** 条件可标为「删除候选」，并在列表中写出对应规则编号。

## 规则

| 编号 | 条件 | 示例 |
|------|------|------|
| R1 | 标题 trim 后长度 ≤ 2 | `1`、`ab` |
| R2 | 标题仅为纯数字 | `1`、`123` |
| R3 | 标题与描述相同，且长度 ≤ 10 | title=`测试`, description=`测试` |
| R4 | 描述 trim 后长度 ≤ 4，且标题无具体语义 | title=`1`, description=`测试` |
| R5 | 标题或描述含「测试反馈」「测试 Hermes」「test@example.com」等明显联调/邮件测试用语 | Hermes 邮件测试条目 |
| R6 | 标题为泛词且无有效描述：`测试`、`test`、`asdf`、`aaa` 等 | — |

## 非无效（默认保留）

- 标题短但描述有具体复现（如「闪退」+ 多步描述）
- 已 `fixed` 且有 solution 的历史 bug（除非用户明确要求清历史垃圾）
- `category` 不是 `bug`（清除无效 **bug** 模式不删 feature/question）

## 输出格式（删除候选列表）

```markdown
| id | title | 规则 | 理由 |
|----|-------|------|------|
| fb_20260609_002 | 1 | R1,R2,R4 | 标题为数字且无有效描述 |
```

用户确认时可按 id 全选或 subset。
