## P2 · Agent 构建（下一阶段）⏭

**目标**：将现有 `Agent.ts` + MCP + Skill + HITL 串成稳定、可观测、可扩展的智能体运行时。

### 2.1 核心运行时

- [ ] 统一 Agent 回合状态机（工具调用 → 结果回灌 → 继续/结束）
- [ ] 与 `LlmClient` 流式、取消、错误分类对齐
- [ ] 工具调用步骤 UI 与日志可追溯（requestId / toolCallId）
- [ ] Skill 在 agent 模式下的自动加载策略与 excludedSkills 一致性

### 2.2 工具与 MCP

- [ ] MCP 工具注册表与 Agent 工具命名空间合并策略
- [ ] HITL 批准流默认策略（设置页已有开关，需与 Agent 路径全覆盖）
- [ ] 本地工具：读文件、笔记检索、网页摘要等优先级排序

### 2.3 质量与测试

- [ ] Agent 回合集成测试（mock LLM + mock tools）
- [ ] 回归：对话模式 / 阅读模式 / Agent 模式互不干扰
- [ ] 文档：Agent 架构说明与扩展指南

### 2.4 交付里程碑

| 里程碑 | 说明 |
|--------|------|
| **M2.1** | Agent 单轮工具调用稳定可用 |
| **M2.2** | 多轮工具 + MCP 联调通过 |

## P3 · 生成模式优化（已纳入计划）

生成模式当前为**雏形**：全屏纸张 + 透明输入层 + Markdown 实时渲染。后续按语雀式体验迭代。

### 3.1 编辑体验

- [ ] 光标与渲染层像素级对齐（行高、标题块、列表缩进）
- [ ] 块级编辑：Enter 分段、Backspace 合并
- [ ] 浮动格式工具栏（粗体 / 标题 / 列表 / 链接）
- [ ] 可选：源码模式切换（Monaco 或轻量 textarea）

### 3.2 文档与持久化

- [ ] 多草稿 / 文档列表（非单一 `__generate__` 键）
- [ ] 导出 `.md`、另存为项目文件
- [ ] 与笔记本 / AI 生成结果对接（插入、续写）

### 3.3 AI 协同

- [ ] 生成模式内唤起 AI：选区续写、全文润色、大纲扩写
- [ ] Agent 只读/写入生成草稿的专用工具

### 3.4 视觉

- [x] 纸张铺满整页（已完成）
- [ ] 可选窄栏阅读宽度切换
- [ ] 暗色主题适配
